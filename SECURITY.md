# Security Policy

Security issues should be reported as a private issue through the repository's
Security advisories so they can be fixed before public disclosure.

## Reporting

1. Open a private advisory on GitHub.
2. Include a short description and impact assessment.
3. Provide reproduction steps and affected versions, if known.
4. Do not include exploitation details in public comments.

## Scope

This project currently targets iOS and distributes through source package usage.

When reporting, include:

- Environment (macOS, Xcode, iOS versions)
- Reproduction steps
- Any attached sample markdown text or code snippets
